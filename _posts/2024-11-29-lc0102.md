LPworks 데이터 전송에서는 조합별로 조합의 투자 정보를 LPworks로 전송하는 방법과, LPworks로부터 수신한 데이터 전송 요청을 확인하고 처리하는 방법을 설명합니다.

## 설명

### 화면 경로
`VCworks` → `LP보고` → `LPworks` → `데이터 전송`

### 조합 & 데이터 선택하여 전송하기
LPworks를 사용하는 출자자에게 조합별로 전송하고자 하는 정보를 선택하여 전송하는 방법은 다음과 같습니다.

1. `1. 대상 조합 선택`에서 정보를 전송하고자 하는 조합을 선택합니다.
	- `청산 조합 표시` 토글을 통해, `청산완료` 조합도 선택할 수 있습니다.
	- `최근 전송일시` 셀을 클릭하면 해당 조합이 필터링된 상태로 `LPworks 전송 이력` 화면으로 이동합니다.
2. `2. 전송 데이터 선택`에서 전송하고자 하는 정보를 선택합니다.
	- 각 항목별 전송 데이터
		- 조합 정보: 조합의 기본 정보, 운용 인력, 회계 감사인 정보를 전송합니다. 필수 항목입니다.
		- 출자/배분 및 지분양수도: 조합의 출자/배분 및 지분양수도 거래 내역을 전송합니다. 필수 항목입니다.
		- 출자/배분 계획: 조합의 보고기준일별 출자/배분 계획을 선택하여 전송합니다.
		- 조합 수익률: 조합의 수익률 정보를 전송합니다.
		- 조합 재무제표: 조합의 메인 회계원장에 기록된 재무상태표와 손익계산서 데이터, 이를 기반으로 생성된 재무제표 파일을 전송합니다.
		- 투자심사 보고: 조합의 본투자심의 건 중 보고할 건을 선택하여 전송합니다.
		- 영업보고서: 조합별로 등록한 월/분기/수시 영업보고서 파일을 전송합니다.
		- 포트폴리오 정보: 조합이 투자한 포트폴리오 기본 정보를 전송합니다. 필수 항목입니다.
		- 포트폴리오 투자현황: 조합이 투자한 포트폴리오의 투자현황 정보를 전송합니다. 필수 항목입니다.
		- 포트폴리오 가치평가: 조합이 투자한 포트폴리오의 가치평가 정보를 전송합니다.
		- 포트폴리오 수익률: 조합이 투자한 포트폴리오의 수익률 정보를 전송합니다.
		- 포트폴리오 재무제표: 조합이 투자한 포트폴리오의 재무제표 정보를 전송합니다.
		- 포트폴리오 사업현황: 분기별 영업보고서에 작성한 사업 현황 정보를 전송합니다.
	- 자동 선택된 항목은 다른 데이터 전송에 필수적인 기초 정보로 함께 전송됩니다.
	- 전송 전 추가행동이 필요한 항목은 해당 항목을 클릭하여 대상 또는 기준일을 먼저 저장해야 합니다.
3. `데이터 전송`을 클릭해 주세요.
4. `LPworks 데이터 전송` 컨펌 모달에서 전송 대상 조합 수를 확인한 뒤 [확인]을 클릭해 주세요.
	- [tip] 데이터 전송까지 약 5~10분 가량이 소요됩니다.

#### 전송 전 추가행동
#### 출자/배분 계획 전송 대상 선택
출자/배분 계획 전송 시에 전송하고자 하는 계획 건을 선택하여 전송할 수 있습니다.

1. `2. 전송 데이터 선택`에서 출자/배분 계획 `>` 아이콘을 클릭해주세요.
2. `출자/배분 계획 전송 대상 선택` 모달에서 전송할 계획 건의 체크박스를 선택해주세요.
	- `1. 대상 조합 선택`에서 선택한 조합이 포함된 계획 건만 표시됩니다.
	- LPworks 전송 현황이 `재전송 필요` 또는 `미전송`인 계획 건은 모달 진입 시 기본 선택됩니다.
3. `저장` 버튼을 눌러주세요.
	- 선택한 계획 건 수가 전송 데이터 항목에 표시되고, 데이터 전송 시 선택한 계획 건만 LPworks로 전송됩니다.

##### 출자/배분 계획의 LPworks 전송 현황

| 상태     | 설명                                                                 |
| ------ | ------------------------------------------------------------------ |
| 완료     | 해당 계획 건의 LPworks 전송 성공 이력이 있고, 가장 최근 전송 성공 이후 수정 이력이 없는 경우를 의미합니다. |
| 미전송    | 해당 계획 건의 LPworks 전송 성공 이력이 없는 경우를 의미합니다.                           |
| 재전송 필요 | 해당 계획 건의 LPworks 전송 성공 이력이 있고, 가장 최근 전송 성공 이후 수정 이력이 있는 경우를 의미합니다. |

#### 조합 재무제표 기준일 선택
조합 재무제표 전송 시에 기준일자를 선택하여 전송할 수 있습니다.

1. `2. 전송 데이터 선택`에서 조합 재무제표 `>` 아이콘을 클릭해주세요.
2. 기준일 선택 드로어에서 기준일자를 선택해주세요.
3. `저장` 버튼을 눌러주세요.
	- 저장한 기준일자가 전송 데이터 항목에 표시됩니다.
	- 데이터 전송 시 해당 조합의 재무제표 기준일은 선택된 기준일을 바탕으로 다음과 같이 조회 및 첨부파일을 생성하여 전송합니다.
		- 재무상태표 기준일
			- 당기 조회일: 선택한 기준일
			- 전기 조회일: 선택한 기준일의 전 회계연도 말일
		- 손익계산서 기준기간
			- 당기: 조합 회계연도 초일부터 기준일까지
			- 전기: 이전 회계연도 전체 기간
		- `인감포함`으로 출력된 재무상태표와 손익계산서 PDF 파일이 함께 전송됩니다.

#### 투자심사 보고 대상 선택
투자심사 보고 전송 시에 전송하고자 하는 투자심사 건을 선택하여 전송할 수 있습니다.

1. `2. 전송 데이터 선택`에서 투자심사 보고 `>` 아이콘을 클릭해주세요.
2. 투자심사 보고 전송 대상 선택 모달에서 전송하고자 하는 건의 체크박스를 선택해주세요.
	- 심의 단계가 `본투자심의` 이상이며, 선택 조합에 대한 심의가 개최된 건만 표시됩니다.
	- 계획보고가 `가능`이거나 결과보고가 `가능`인 투자심사 건은 모달 진입 시 기본 선택됩니다.
	- `투자기업명` 셀을 클릭하여, 해당 투자심사 건의 등록 상세 페이지로 진입할 수 있습니다.
	- 표시된 투자심사 건 중 다음 조건을 모두 만족하는 투자심사 건만 체크박스 선택이 가능합니다.
		- [회사정보] 탭의 `국가`, `사업자번호`, `표준산업분류`, `자본금`, `설립일자` 모두 입력
		- [담당심사역] 탭의 `담당자` 입력
		- [투자정보] 탭의 투자조건, 투자 라운드 정보, 기타 정보 필수값 모두 입력
			- 동반 투자 조건이 있는 경우, 동반 투자 개요와 조건의 필수값 모두 입력
		- [본투자심의] 탭의 재원별 `개최일시`와 `장소`가 입력된 개최 정보 존재
			- 결과보고인 경우, 투표결과가 `가결` 또는 `부결`
		- [첨부파일] 탭의 `투자심사보고서`, `준법사항체크리스트`, `투심위의사록`(결과보고 시에만) 모두 첨부
3. `저장` 버튼을 눌러주세요.
	- 데이터 전송 시 체크박스가 선택된 건들만 LPworks로 전송됩니다.
		- 결과보고 값이 없는 건은 계획보고 값이 `가능` 혹은 `완료`인 경우, 계획보고로 분류되어 전송됩니다.
		- 결과보고 값이 있는 건은 결과보고 값이 `가능` 혹은 `완료`인 경우, 결과보고로 분류되어 전송됩니다.

##### 투자심사 보고 상태

| 상태  | 설명                                             |
| --- | ---------------------------------------------- |
| 가능  | 현재 보고단계에서 전송 조건을 모두 만족하여 보고할 수 있는 상태입니다.       |
| 불가능 | 현재 보고단계에서 전송 조건을 하나 이상 만족하지 못해 보고할 수 없는 상태입니다. |
| 완료  | 해당 보고단계의 보고가 이미 이뤄진 상태입니다.                     |
| 만료  | 투심결과 단계에 도달했지만 계획보고가 이뤄지지 않은 상태입니다.            |

#### 조합별 영업보고서 첨부
한글 또는 word파일 등으로 생성한 조합별 영업보고서를 직접 첨부하여 전송할 수 있습니다.

1. `2. 전송 데이터 선택`에서 영업보고서 `>` 아이콘을 클릭해주세요.
2. 영업보고서 등록 드로어에서 파일을 첨부할 조합을 클릭해주세요.
3. 기준일과 분기를 선택하신 후, 영업보고서를 첨부해주세요.
	- 하단의 버튼을 통해 행을 제거하거나 추가할 수 있습니다.
4. `저장` 버튼을 눌러주세요.
	- 데이터 전송 시 `영업보고서` 항목을 선택한 경우, 첨부한 파일이 LPworks로 전송됩니다.


### LPworks 데이터 전송 요청 처리하기
LPworks의 조합원이 데이터 전송을 요청한 경우, `요청 현황`에서 요청 목록을 확인하고 필요한 데이터를 전송할 수 있습니다.

1. 상단의 [요청 현황] 버튼을 클릭해 주세요.
	- LPworks 데이터 전송 요청 알림을 클릭해도 `데이터 전송 요청 현황` 모달로 이동할 수 있습니다.
2. `데이터 전송 요청 현황` 모달에서 요청 일시, 요청 조합원, 대상 조합, 요청 데이터, 요청 기한, 처리 현황을 확인해 주세요.
	- 처리 현황이 `미처리`인 요청 행은 모달 진입 시 자동 선택됩니다.
	- 처리 현황이 `완료` 또는 `전송중`인 요청 행은 선택할 수 없습니다.
	- 요청 데이터가 2개 이상인 경우, 축약 표시되며 셀을 클릭해 전체 요청 데이터 목록을 확인할 수 있습니다.
3. 전송할 요청 행을 선택한 뒤 [데이터 전송] 버튼을 클릭해 주세요.
	- 선택된 요청 행에 포함된 {대상 조합-요청 데이터}를 중복 없이 LPworks로 전송합니다.

##### 요청 처리 현황
데이터 전송 요청 현황의 `처리 현황`은 다음 중 하나의 값을 가지게 됩니다.

| 상태  | 설명                                                                                            |
| --- | --------------------------------------------------------------------------------------------- |
| 완료  | 해당 요청에 포함된 모든 [대상 조합-요청 데이터]에 대해, 요청 일시 이후의 전송 이력 중 전송 결과가 `성공`인 경우가 존재함을 의미합니다.              |
| 미처리 | 해당 요청에 포함된 모든 [대상 조합-요청 데이터]에 대해, 요청 일시 이후의 전송 이력 중, 전송 결과가 `성공`인 이력이 하나라도 존재하지 않는 경우를 의미합니다. |
| 전송중 | `미처리` 중 전송 결과가 `전송중`인 이력이 하나라도 존재하는 경우를 의미합니다.                                                |

### LPworks 전송 정책
LPworks로 전송되는 데이터는 전송 항목의 유형에 따라 전송 단위가 다릅니다.

- `LPworks 데이터 전송` 화면에서만 전송할 수 있는 항목은 해당 항목에 누적되어 등록된 전체 행을 기준으로 전송됩니다.
- 각 항목을 등록한 VCworks의 특정 화면에서 전송할 수 있는 항목은 해당 화면의 행 단위로 전송됩니다.
- 동일한 전송 단위에 대한 복수 전송은 동시에 여러 건으로 처리되지 않습니다.
- 동일한 전송 단위에 대한 전송 상태 및 이력은 모든 전송 가능 화면에 같은 값을 공유합니다.

## 자주 묻는 질문

> LPworks로 잘 전송됐는지 확인할 수 있는 방법이 있나요?
{: .prompt-tip }
- LPworks에 대한 모든 전송 이력은 `LPworks 전송 이력` 화면에서 확인할 수 있습니다.
	- `VCworks` → `LP보고` → `LPworks` → `전송 이력`
- `대상 조합 선택`의 `최근 전송일시`를 클릭하면 해당 조합이 필터링된 상태로 `LPworks 전송 이력` 화면으로 이동합니다.

> LPworks에서 요청한 데이터 전송 요청은 어디서 확인하나요?
{: .prompt-tip }
- 상단의 `요청 현황` 버튼을 클릭해 `데이터 전송 요청 현황` 모달에서 확인할 수 있습니다.
- 미처리 요청 행은 모달 진입 시 자동 선택되며, [데이터 전송] 버튼으로 요청 데이터를 전송할 수 있습니다.

> 대상 조합을 여러 개 선택하면 어떻게 전송되나요?
{: .prompt-tip }
- 1에서 선택한 각 조합에 대해, 2에서 선택한 모든 데이터가 각각 전송됩니다.
  (ex. 3개 조합 x 5개 데이터 → 총 15건 전송)
- LP는 본인이 출자한 조합의 정보만 조회할 수 있습니다.

> 전송항목에 대해 동일한 식별자로 여러번 전송하면 어떻게 되나요?(ex. 동일 조합-보고기준일의 출자/배분 계획 전송)
{: .prompt-tip }
- LPworks에서 LP는 기본적으로 동일한 전송 항목에 대해, 가장 최신 전송된 값만 조회합니다.
- 동일한 전송 단위의 전송 상태와 이력은 전송 가능한 화면들에서 같은 값으로 공유됩니다.

> 전송 중인 항목이 있다고 나와요.
{: .prompt-tip }
- 선택한 대상 조합의 전송 데이터 중 이미 전송중인 항목이 있는 경우입니다.
- 본 화면에서 전송하지 않았더라도 각 모듈별 전송화면에서 해당 항목에 대한 전송이 진행 중인 경우, 전송이 제한됩니다.

> 전송을 취소할 수는 없나요?
{: .prompt-tip }
- 현재 이미 전송 요청된 건을 취소할 수 있는 기능은 제공하지 않고 있습니다.
