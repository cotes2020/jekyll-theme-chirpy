# 상세 제안서 - VICS 데이터 기반 공정가치평가 자료 제출 표준화

> NICE P&I·KIS자산평가 평가자료 수신 협력안  
> 작성: 똑똑 주식회사 / 기준일: 2026년 5월 28일

## 1. 제안 요약

똑똑은 NICE P&I와 KIS자산평가가 2026년 7월 공정가치평가 업무에서 운용사 자료를 더 표준화된 방식으로 수신·검토할 수 있도록, VICS 데이터와 VCworks를 활용한 공정가치평가 자료 생성·보완·제출 흐름을 제안드립니다.

이번 제안의 핵심은 운용사가 이미 VICS에 보고해 온 데이터를 공정가치평가 제출자료 준비의 출발점으로 활용하는 것입니다. VICS 데이터로 자동 생성 가능한 항목은 먼저 채우고, 부족 항목만 운용사가 보완하게 하면 평가기관은 더 표준화된 자료를 수신할 수 있고, 운용사는 자료 제출 부담을 줄일 수 있습니다.

- 대상 평가기관은 NICE P&I와 KIS자산평가입니다.
- 대상 운용사는 2026년 6월 30일 기준 공정가치평가 대상 운용사 중 VCworks 미사용 또는 부분 사용 운용사입니다.
- 적용 시점은 2026년 7월 평가자료 준비 및 제출 흐름부터로 봅니다.
- 기본 방식은 VICS 데이터 역다운로드, 기존 운용사 자료, VCworks 입력 화면을 결합해 평가기관 제출자료를 생성하는 것입니다.
- 전달 방식은 초기에는 표준 파일 묶음 또는 협의된 전달 경로로 제출하고, 이후 필요 시 전송 방식을 고도화합니다.
- 목표는 운용사 자료 제출 부담 감소, 평가기관 자료 수신 표준화, 누락·재요청 감소, 평가 이후 데이터 재활용입니다.

제안의 기본 방향은 두 평가기관에 동일합니다. 평가기관별 요구 양식이나 필수 항목은 일부 다를 수 있으므로, 공통 원천 데이터와 공통 입력 흐름은 VCworks에서 통합 관리하고, 최종 제출 파일·필수값·첨부자료만 평가기관별로 분기하는 방식으로 운영합니다.

## 2. 제안 배경

모태출자조합의 공정가치평가 업무가 확대되면서 운용사는 평가기관에 제출할 자료를 정리해야 하고, 평가기관은 다수 운용사로부터 받은 자료를 검토·보완·정리해야 합니다. 이 과정이 메일, 엑셀, 첨부파일 중심으로 운영되면 운용사와 평가기관 모두에게 반복적인 비효율이 발생합니다.

운용사 측의 주요 부담은 다음과 같습니다.

- 조합정보, 출자·배분, 투자자산, 재무제표, 첨부파일을 각각 다른 원천에서 취합해야 합니다.
- 평가기관 제출용 엑셀, 내부 관리 파일, VICS 보고, LP 보고에 유사 정보를 반복 입력해야 합니다.
- 평가기관별 양식 차이를 이해하지 못하면 필수 항목 누락이나 값 불일치가 발생할 수 있습니다.
- 제출 완료 후 데이터가 운용사 내부 관리나 다음 보고에 다시 활용되지 못하는 경우가 많습니다.

평가기관 측의 주요 부담은 다음과 같습니다.

- 운용사별 제출 파일 구조, 파일명, 첨부자료 수준이 달라 검토 기준을 통일하기 어렵습니다.
- 필수 항목 누락, 값 불일치, 첨부파일 누락 시 운용사와 여러 차례 재커뮤니케이션이 발생합니다.
- 메일로 받은 엑셀과 첨부파일을 내부 평가 업무에 맞게 다시 구조화해야 합니다.
- 평가 착수 전 자료 보완과 일정 안내에 평가기관 실무 시간이 과도하게 투입됩니다.

VCworks는 운용사의 조합·출자·배분·투자자산·첨부자료를 ERP 원천 데이터로 관리하는 시스템입니다. 여기에 VICS 데이터 역다운로드와 공정가치평가 입력 화면을 결합하면, 운용사가 평가기관에 제출해야 하는 자료를 보다 구조화된 방식으로 준비할 수 있습니다.

## 3. 제안 목표

본 제안의 목표는 다음 네 가지입니다.

1. 운용사가 공정가치평가 제출자료를 VCworks에서 한 번에 준비할 수 있게 합니다.
2. NICE P&I와 KIS자산평가가 운용사 자료를 표준화된 구조로 수신할 수 있게 합니다.
3. VICS 데이터와 VCworks 데이터를 결합해 최초 입력 부담을 줄이고, 부족 항목만 고객이 보완하게 합니다.
4. 평가 이후에도 해당 데이터가 VICS 보고, 영업보고서, 포트폴리오 관리, LP 보고에 재활용되도록 합니다.

중요한 점은 평가기관별 전용 화면을 각각 만드는 것이 아닙니다. VCworks 안에서 공통 원천 데이터를 만들고, 평가기관별 제출 포맷과 필수값만 분기하는 구조를 만드는 것이 핵심입니다.

## 4. 통합 협력 모델

### 4.1 전체 구조

제안하는 협력 모델은 다음 흐름을 기본으로 합니다.

1. 운용사가 VICS 데이터 역다운로드를 요청하거나, 기존에 보유한 VICS·평가자료·관리자료를 준비합니다.
2. 운용사가 VCworks에 접속해 VICS 파일 또는 기존 자료를 업로드합니다.
3. VCworks가 조합정보, 출자·배분, 조합원정보, 투자자산 등 자동 입력 가능한 항목을 생성합니다.
4. 운용사는 VCworks 화면에서 부족 항목만 보완하고 첨부파일을 확인합니다.
5. VCworks가 NICE P&I 또는 KIS자산평가 제출용 자료 묶음을 생성합니다.
6. 평가기관은 표준화된 자료를 수신하고, 누락·보완 요청은 VCworks 기준으로 회신합니다.
7. 평가 완료 후 결과 파일 또는 결과 요약 정보를 운용사에 전달하고, 필요 시 VICS 보고 또는 운용사 내부 관리로 이어갑니다.

### 4.2 주체별 역할

- 운용사: VICS 데이터 확보, VCworks 접속, 부족 항목 보완, 첨부파일 확인, 최종 제출 승인을 담당합니다.
- 똑똑(VCworks팀: FO그룹): 데이터 업로드, 자동 매핑, 부족 항목 식별, 화면 입력, 검증, 제출자료 생성, 제출 이력 관리를 담당합니다.
- NICE P&I: 제출자료 수신 기준 확인, 필수·선택 항목 구분, 파일럿 검수, 운용사 안내 협력을 담당합니다.
- KIS자산평가: 제출자료 수신 기준 확인, 필수·선택 항목 구분, 파일럿 검수, 운용사 안내 협력을 담당합니다.
- 똑똑(사업기획팀): 시스템 운영, 고객 안내자료 제공, 파일럿 운영 지원, 평가기관 피드백 반영을 담당합니다.

### 4.3 평가기관별 분기 원칙

| 구분 | 공통 처리 | 평가기관별 분기 |
|------|-----------|----------------|
| 운용사 입력 화면 | 동일 화면 사용 | 평가기관별 필수 항목 표시 |
| 원천 데이터 | 동일 데이터 모델 사용 | 기관별 제출 대상 컬럼만 선택 |
| 첨부파일 | 동일 업로드·확인 흐름 사용 | 기관별 필수 첨부 체크리스트 분기 |
| 제출 파일 | 동일 생성 엔진 사용 | 파일명, 시트·CSV 구성, 컬럼명, 인코딩 등 분기 |

## 5. VCworks 기반 공정가치평가 업무 흐름

### 5.1 운용사 사용 흐름

평가기관이 공정가치평가 대상 운용사에 VICS 데이터 기반 제출자료 정리 방식을 안내하면, 똑똑은 신청 운용사를 접수하고 VCworks 사용 절차를 안내합니다. 운용사는 VICS 데이터와 기존 평가자료를 VCworks에 업로드해 제출자료를 생성하고, 평가 이후에는 동일 데이터를 VICS 월보고, 영업보고서, BI, 다음 가치평가 준비에 재사용할 수 있습니다.

```{.mermaid height=0.62}
flowchart TD
    A[1. 공문 발송<br/>고객 안내]
    B[2. VICS 데이터<br/>확보·업로드]
    C[3. 평가자료 준비<br/>기준 설정]
    D[4. 기존 자료<br/>첨부파일 추가]
    E[5. 기초 데이터<br/>자동 생성]
    F{6. 필수값·첨부파일<br/>충족?}
    G[부족 항목 보완]
    H[7. 제출자료 생성·승인]
    I[8. 평가기관 수신·평가 착수]
    J[9. 평가 후<br/>VCworks 재사용]

    A --> B --> C --> D --> E --> F
    F -- 부족 --> G --> F
    F -- 충족 --> H --> I --> J
```

주요 단계는 다음과 같습니다.

1. 평가기관 관심 운용사 안내

   평가기관은 공정가치평가 대상 운용사에 자료 요청을 안내하면서, VICS 데이터 기반 제출자료 정리 방식과 VCworks 1년 무상 사용 혜택을 함께 소개합니다. 방식은 평가기관 안내 메일 내 신청 링크 제공, 관심 운용사 추천 명단 전달, 담당자 연결 중 7월 적용 가능한 형태로 정합니다.

2. 똑똑 접수·초기 안내

   똑똑은 평가기관을 통해 유입된 운용사를 접수하고, VCworks 사용 계정, 운용사용 퀵가이드, VICS 역다운로드 공문 양식, 제출자료 준비 범위를 안내합니다. 이 단계의 목적은 운용사가 “무엇을 어디에 올리면 되는지”를 빠르게 이해하게 하는 것입니다.

3. VICS 데이터 확보·VCworks 업로드

   운용사는 안내받은 절차에 따라 VICS 데이터를 역다운로드하고, 기존 평가자료와 첨부파일을 VCworks에 업로드합니다. VCworks는 조합, 출자, 배분, 투자자산 관련 데이터를 제출자료 생성의 기초 자료로 정리합니다.

4. 공정가치평가 제출자료 생성

   운용사는 VCworks에서 평가기관, 평가기준일, 대상 조합을 선택하고 부족 항목만 보완합니다. VCworks는 7월 협의된 필수 항목과 파일 구성에 맞춰 NICE P&I 또는 KIS자산평가 제출자료 묶음을 생성합니다.

5. 평가기관 수신·평가 착수

   평가기관은 협의된 파일 형식과 전달 경로로 자료를 수신합니다. 초기에는 완전한 시스템 연동보다 7월 평가에 바로 사용할 수 있는 표준 파일 묶음 전달을 우선합니다.

6. 평가 이후 VCworks 운영 전환

   평가 완료 후 운용사는 제출 과정에서 정리된 데이터를 VCworks에 남겨 이후 운영 데이터로 재사용합니다. 이때 VICS 월보고를 VCworks에서 수행하고, 포트폴리오 관리와 가치평가 데이터 관리로 확장할 수 있습니다.

7. 영업보고서·BI·다음 가치평가 재사용

   한 번 정리된 데이터는 영업보고서 자동화, BI·리포트 생성, LP 보고, 다음 평가기준일의 공정가치평가 자료 생성에 재사용합니다. 따라서 운용사 입장에서 본 제안은 7월 평가자료 제출 지원에 그치지 않고, VCworks를 실제 운영 데이터로 검증하는 진입 흐름이 됩니다.

### 5.2 화면 구성 기준

VCworks 공정가치평가 화면은 다음 탭 구조를 기준으로 구성합니다.

1. 기본정보

   조합명, 평가기관, 평가기준일, 마감일을 관리합니다. 일부 정보는 기존 데이터로 구성할 수 있으나, 평가기관과 기준일은 운용사가 제출 목적에 맞게 선택해야 합니다.

2. 조합정보

   펀드명, 유형, 약정총액, 설립일, 존속기간, 성과보수 조건을 관리합니다. VICS와 VCworks 원천 데이터를 우선 활용하고, 규약 기반 조건처럼 원문 확인이 필요한 항목은 운용사가 보완합니다.

3. 조합 재무제표

   재무상태표와 손익계산서를 관리합니다. VCworks 회계 원천을 사용할 수 있는 운용사는 해당 데이터를 활용하고, 미사용 운용사는 엑셀 업로드, OCR, 직접 입력 중 7월 적용 가능한 방식으로 처리합니다.

4. 출자배분 현황

   출자, 배분, 조합원별 납입·분배 내역을 관리합니다. VICS 데이터 활용 가능성이 높으므로 자동 생성 후 예외 케이스를 확인하는 흐름으로 구성합니다.

5. 투자조건별 현황

   최초 투자조건, 주식종류, 단가, 잔액을 관리합니다. 투자원장과 VICS 데이터를 우선 활용하되, Refixing, PUT/CALL, 장부단가처럼 해석이 필요한 항목은 운용사 확인이 필요합니다.

6. 투자자산 현황

   평가기준일 기준 투자자산 현황과 투자·회수·전환 내역을 관리합니다. 상당 부분은 기존 거래 데이터로 구성할 수 있으며, 잔액 불일치 사유와 일부 상세값은 운용사가 보완합니다.

7. 제3자 거래 현황

   제3자 유상증자, 거래 단가, 거래일을 관리합니다. VCworks나 VICS에 원천 데이터가 없는 경우가 많으므로 운용사가 직접 입력하거나 증빙을 첨부합니다.

8. 첨부파일

   규약, 투자계약서, 제3자 거래 증빙 등 제출자료에 필요한 파일을 관리합니다. 일부 파일은 기존 관리 자료를 활용하고, 누락 파일은 운용사가 업로드합니다.

### 5.3 데이터 충족률에 대한 현재 판단

현재 PoC 기준으로 VICS 데이터는 공정가치평가 제출자료의 상당 부분을 자동 생성할 수 있습니다. 다만 모든 평가 항목을 자동으로 완성하는 수준은 아니며, 운용사의 직접 보완과 평가기관 협의가 함께 필요합니다.

| 데이터 영역 | 현재 판단 | 처리 방향 |
|-------------|-----------|-----------|
| 조합정보 | 50~80% 수준 | VICS·VCworks 원천으로 생성 후 규약 기반 조건 보완 |
| 조합 재무제표 | 별도 처리 필요 | 회계 원천, 엑셀 업로드, OCR, 직접 입력 중 적용 |
| 출자배분 현황 | 100% 수준 가능 | VICS 데이터 기반 자동 생성 |
| 투자조건별 현황 | 약 70% 수준 | VICS·투자원장 기반 생성 후 옵션·상장여부 등 보완 |
| 투자자산 현황 | 약 85% 수준 | 투자·회수·전환 거래 기반 생성 후 불일치 사유 보완 |
| 제3자 거래 | 원천 없음 | 운용사 직접 입력 또는 증빙 첨부 |
| 첨부파일 | 원천별 차이 큼 | 규약·계약서·증빙 파일 업로드 안내 |

VCworks는 VICS 데이터와 기존 자료로 먼저 채울 수 있는 항목을 생성하고, 운용사가 부족 항목만 보완하도록 안내합니다. 평가기관 제출자료는 7월 협의된 필수 항목과 파일 구성에 맞춰 생성하는 것을 우선합니다.

## 6. 평가기관 제공 가치

### 6.1 자료 수신 표준화

VCworks를 통해 제출되는 자료는 동일한 화면, 동일한 검증 기준, 동일한 파일 생성 절차를 거칩니다. 이를 통해 평가기관은 운용사별로 다른 파일명, 시트명, 컬럼명, 첨부자료 구성 때문에 발생하는 검토 비용을 줄일 수 있습니다.

적용 전후 변화는 다음과 같습니다.

- 운용사별 메일·엑셀 양식이 달랐던 부분은 평가기관별 표준 포맷 생성 방식으로 정리합니다.
- 첨부파일 누락 여부를 수작업으로 확인하던 부분은 제출 전 체크리스트로 확인합니다.
- 필수 항목 누락 시 개별 재요청하던 부분은 입력 단계의 필수값 검증으로 줄입니다.
- 평가기관 내부에서 다시 정리하던 부분은 구조화된 자료 수신으로 평가 착수 전처리를 줄입니다.

### 6.2 재요청 감소

운용사가 제출 버튼을 누르기 전, VCworks는 평가기관별 필수값과 첨부파일 상태를 점검합니다. 이 구조를 통해 평가기관이 평가 착수 전에 발견하는 누락 항목을 줄일 수 있습니다.

검증 예시는 다음과 같습니다.

- 필수값: 펀드명, 평가기준일, 약정총액, 투자자산명, 투자잔액을 확인합니다.
- 합계 검증: 조합별 출자합계, 배분합계, 투자잔액 합계를 확인합니다.
- 기준일 검증: 평가기준일 이후 거래가 포함되었는지 확인합니다.
- 첨부파일 검증: 규약, 투자계약서, 제3자 거래 증빙의 첨부 여부를 확인합니다.
- 불일치 사유: 장부잔액과 투자잔액이 일치하지 않을 때 사유 입력 여부를 확인합니다.

### 6.3 운용사 고객 응대 효율화

평가기관은 운용사에 VICS 데이터 기반 제출자료 정리 방식을 안내함으로써 자료 요청과 보완 커뮤니케이션을 단순화할 수 있습니다. 평가기관이 고객에게 모든 입력 방식을 직접 설명하는 것이 아니라, VCworks 퀵가이드와 화면 안내를 활용하게 하는 방식입니다.

운용사는 VCworks 화면에서 “무엇이 부족한지”를 확인하고, 평가기관은 “어떤 항목이 평가기관 필수값인지”를 일관된 기준으로 회신할 수 있습니다.

### 6.4 평가 이후 확장성

운용사가 공정가치평가를 위해 정리한 데이터는 다음 업무로 확장될 수 있습니다.

- VICS 월보고 또는 수시보고 준비
- 운용사 내부 포트폴리오 관리
- LP 보고 또는 영업보고서 작성
- 후속 평가기준일의 공정가치평가 자료 재생성
- 후속 기준일 제출자료 재사용

평가기관 입장에서는 고객이 동일 데이터를 반복적으로 정리할 수 있게 되어, 다음 평가 라운드의 자료 수신 품질도 개선될 수 있습니다.

## 7. 운용사 고객 제공 가치

운용사에는 “공정가치평가 제출자료를 준비하는 과정이 곧 VCworks 도입 체험이 됩니다”라는 메시지로 안내할 수 있습니다. VCworks 1년 무상 사용에 따른 기능별 효익은 [별첨 1. VCworks 1년 무상 사용 고객 혜택](/assets/files/vics-fair-value-proposal/appendix-1-vcworks-benefits.pdf)에 정리합니다.

핵심 고객 효익은 다음 네 가지입니다.

- 제출 준비 시간 단축: VICS 데이터와 기존 평가자료를 활용해 조합·출자·배분·투자자산 정보를 먼저 채우고, 운용사는 부족 항목만 보완합니다.
- 누락·재요청 감소: 평가기관 제출 전에 필수값과 첨부파일 누락을 화면에서 확인하므로, 평가 착수 전 보완 커뮤니케이션을 줄일 수 있습니다.
- 1년 무상 사용으로 실제 업무 개선 체험: 공정가치평가 데이터를 기반으로 BI·리포트, 영업보고서 자동화, 투자·회수 관리, 가치평가 데이터화를 실제 업무에서 확인할 수 있습니다.
- LP·투자기업까지 이어지는 확장 효과: VCworks 사용 고객의 LP는 LPworks를, 투자기업은 STworks를 함께 활용할 수 있어 자료 조회·보고·소통 업무까지 개선할 수 있습니다.

따라서 운용사 입장에서 본 제안은 단순한 제출 편의가 아니라, 공정가치평가를 계기로 VCworks의 핵심 기능과 이해관계자 연결 효과를 1년간 비용 부담 없이 검증할 수 있는 기회입니다.

## 8. 2026년 7월 적용 계획

### 8.1 적용 범위

2026년 7월 적용은 전체 자동화 완성보다 실제 평가자료 준비에 바로 사용할 수 있는 최소 운영 흐름을 우선합니다.

- 평가기관: NICE P&I와 KIS자산평가를 대상으로 합니다.
- 고객 범위: 평가기관이 안내 가능한 관심 운용사 및 내부 파일럿 운용사를 우선 대상으로 합니다.
- 기준일: 2026년 6월 30일 기준 공정가치평가 자료 준비 흐름에 적용합니다.
- 데이터 원천: VICS 데이터, 기존 평가사 제출자료, VCworks 입력 화면, 첨부파일을 활용합니다.
- 산출물: 평가기관 제출자료 묶음, 부족 항목 목록, 제출 이력을 생성합니다.
- 운영 방식: 파일럿 고객부터 순차 적용한 뒤 설명회와 고객 안내로 확장합니다.

### 8.2 일정안

| 시점 | 주요 업무 | 산출물 |
|------|-----------|--------|
| 2026년 5월 말 | NICE P&I·KIS자산평가 통합 제안 및 적용 범위 확인 | 통합 제안서, 평가기관 확인사항 |
| 2026년 6월 1주 | VICS 역다운로드 절차, 운용사 안내문, 제출자료 체크리스트 정리 | 운용사 퀵가이드 초안, 공문 양식 초안 |
| 2026년 6월 2주 | 내부 운용사 또는 관계사 기준 파일럿 데이터 검증 | 자동 생성률, 부족 항목 목록 |
| 2026년 6월 3~4주 | 평가기관별 필수값·제출 포맷 확정, 고객 안내 준비 | 기관별 제출 포맷, FAQ 초안 |
| 2026년 7월 초 | NICE P&I·KIS자산평가 관심 운용사 대상 파일럿 착수 | 파일럿 고객 목록, 제출 테스트 결과 |
| 2026년 7월 중 | 고객 보완 입력, 평가기관 수신·검수 피드백 반영 | 보완 목록, 운영 이슈 목록 |
| 2026년 7월 말 | 공정가치평가 설명회 또는 후속 안내 | 설명회 자료, 파일럿 지원 안내 |

### 8.3 7월 적용의 성공 기준

- 운용사가 VICS 데이터 또는 기존 자료를 VCworks에 업로드할 수 있는지는 파일럿 고객의 업로드 성공 여부로 판단합니다.
- VCworks가 주요 평가자료를 자동 생성하는지는 조합·출자배분·투자자산별 자동 생성률로 확인합니다.
- 부족 항목이 명확히 식별되는지는 화면별 누락 항목 목록 생성 여부로 판단합니다.
- 평가기관이 수신 가능한 자료 묶음이 생성되는지는 NICE P&I와 KIS자산평가의 검수 피드백으로 확인합니다.
- 고객 입력 부담이 기존 방식보다 과도하지 않은지는 파일럿 고객 인터뷰와 보완 항목 수로 판단합니다.
- 평가 이후 동일 데이터 재사용 가능성이 확인되는지는 후속 기준일 자료 재생성 가능성과 추가 조합 적용 의향으로 판단합니다.

## 9. 평가기관에 요청드리는 확인 사항

통합 제안의 실행을 위해 NICE P&I와 KIS자산평가에 다음 사항을 확인드리고자 합니다.

### 9.1 자료 수신 기준

- 제출 단위: 조합별 제출인지, 운용사별 묶음 제출인지 확인이 필요합니다.
- 평가기준일: 2026년 6월 30일 기준 자료 수신 일정을 확인해야 합니다.
- 필수 항목: 평가 착수를 위해 반드시 필요한 항목을 구분해야 합니다.
- 선택 항목: 있으면 좋은 항목과 없어도 평가 착수 가능한 항목을 구분해야 합니다.
- 첨부파일: 규약, 투자계약서, 제3자 거래 증빙 등 필수 첨부 범위를 확인해야 합니다.
- 재요청 기준: 누락 또는 불일치가 있을 때 평가기관이 재요청하는 기준을 확인해야 합니다.

### 9.2 제출 포맷

- 파일 형식: CSV, XLSX, ZIP 등 수신 선호 형식을 확인해야 합니다.
- 시트·파일 구성: 조합정보, 재무제표, 출자배분, 투자자산, 첨부파일 분리 기준을 확인해야 합니다.
- 컬럼명: 평가기관 내부 업무에서 사용하는 컬럼명 또는 표준 양식을 확인해야 합니다.
- 인코딩: CSV 사용 시 UTF-8, CP949 등 인코딩 기준을 확인해야 합니다.
- 파일명: 운용사명, 조합명, 평가기준일, 생성일시 포함 여부를 확인해야 합니다.
- 전달 경로: 이메일, SFTP, 다운로드 링크, 별도 포털 중 7월 적용 가능한 방식을 확인해야 합니다.

### 9.3 파일럿 운영

- 대상 고객: 7월 초 파일럿으로 안내 가능한 운용사 후보를 확인해야 합니다.
- 고객 안내: 평가기관 안내 메일에 VICS 데이터 기반 제출자료 정리 방식과 VCworks 1년 무상 사용 혜택을 포함할 수 있는지 확인해야 합니다.
- 검수 피드백: 파일럿 자료 수신 후 피드백 담당자와 회신 주기를 확인해야 합니다.
- 보완 절차: 운용사 보완 요청을 평가기관이 직접 할지, 똑똑이 중계할지 확인해야 합니다.
- 설명회: 7월 말 전후 공동 설명회 또는 고객 안내 세션 참여 가능 여부를 확인해야 합니다.

## 10. 제출자료 구성안

### 10.1 자료 묶음 구조

초기 제출자료는 다음 묶음으로 구성합니다. 실제 파일명과 시트명은 평가기관별 확인 후 확정합니다.

1. 기본정보: `01_기본정보`

   운용사, 조합, 평가기준일, 평가기관, 담당자 정보를 담습니다. 전체 제출자료를 식별하는 기준 정보이므로 모든 파일과 첨부자료가 이 정보에 연결되도록 구성합니다.

2. 조합정보: `02_조합정보`

   펀드명, 약정총액, 설립일, 존속기간, 투자기간, 보수조건을 담습니다. VICS와 VCworks에 이미 있는 조합 기본정보를 우선 사용하고, 규약 확인이 필요한 조건은 보완 대상으로 분리합니다.

3. 조합 재무제표: `03_조합재무제표`

   재무상태표, 손익계산서, 전기 비교 정보를 담습니다. VCworks 회계 데이터를 활용할 수 없는 운용사는 기존 결산 파일, 평가 제출 파일, 별도 엑셀 업로드 방식으로 보완합니다.

4. 출자배분: `04_출자배분`

   조합원별 출자, 납입, 배분, 잔액을 담습니다. VICS 데이터 활용도가 높은 영역이므로 7월 적용 시 가장 먼저 표준화 효과를 확인할 수 있는 자료 묶음입니다.

5. 투자조건: `05_투자조건`

   투자조건별 주식종류, 단가, 수량, 옵션, 투자잔액을 담습니다. 투자원장과 VICS 데이터를 기준으로 구성하되, 옵션 조건이나 장부단가처럼 해석이 필요한 항목은 보완 기준을 평가기관과 협의합니다.

6. 투자자산: `06_투자자산`

   평가기준일 현재 투자자산, 회수·전환·감액 반영 내역을 담습니다. 운용사가 제출 전 잔액과 변동 내역을 확인할 수 있도록 구성해 평가기관의 사전 검토 부담을 줄입니다.

7. 제3자 거래: `07_제3자거래`

   제3자 유상증자, 거래 단가, 거래일, 관련 증빙을 담습니다. 원천 데이터가 없는 경우가 많으므로 직접 입력 또는 증빙 첨부 방식으로 운영합니다.

8. 첨부파일 목록: `08_첨부파일목록`

   규약, 투자계약서, 제3자 거래 증빙 등 제출자료와 함께 전달해야 하는 파일 목록과 파일명을 담습니다. 평가기관은 첨부파일 수신 여부를 이 목록으로 먼저 확인할 수 있습니다.

9. 검증 결과: `09_검증결과`

   필수값 누락, 합계 불일치, 보완 필요 항목을 담습니다. 운용사가 제출 전 점검할 수 있는 체크 결과이며, 평가기관 입장에서는 재요청 대상을 빠르게 파악하는 기준 자료로 활용할 수 있습니다.

### 10.2 필수·보완·협의 항목 구분

제출자료는 모든 항목을 동일한 강도로 요구하지 않고, 다음 세 단계로 구분합니다.

| 구분 | 의미 | 처리 방향 |
|------|------|-----------|
| 필수 | 평가 착수를 위해 반드시 필요한 항목 | 제출 전 입력 또는 첨부 필수 |
| 보완 | 평가 품질 향상 또는 후속 확인에 필요한 항목 | 누락 시 보완 요청 또는 사유 입력 |
| 협의 | 평가기관별로 요구 수준이 다르거나 기존 메일 방식에서는 원문 확인으로 처리하던 항목 | 7월 파일럿에서 처리 방식 확정 |

이 구분은 고객 입력 부담을 낮추기 위해 중요합니다. 평가기관이 실제 평가 착수에 필요로 하는 항목과, 있으면 좋지만 고객에게 과도한 입력 부담을 만드는 항목을 분리해야 7월 적용이 가능합니다.

## 11. 운용사 자료 제출 지원 방식

### 11.1 고객 안내 메시지

운용사에는 다음 메시지로 안내합니다.

> 공정가치평가 제출자료를 준비할 때 VICS 데이터를 활용하면 조합·출자·배분·투자자산 정보를 먼저 채울 수 있습니다. 부족한 항목만 VCworks 화면에서 보완하면 NICE P&I 또는 KIS자산평가 제출자료를 표준화된 형태로 생성할 수 있습니다.

### 11.2 운용사 단계별 안내

운용사에는 신청, VICS 데이터 확보, VCworks 접속, 파일 업로드, 제출 승인까지의 실행 절차와 필요한 자료를 함께 안내합니다.

| 단계 | 운용사 해야 할 일 | 제공 자료 |
|------|-------------------|-----------|
| 1. 신청 | 공정가치평가 제출자료 정리 지원 신청 | 신청 링크 또는 담당자 안내 |
| 2. VICS 데이터 확보 | 한국벤처투자에 VICS 데이터 역다운로드 요청 | 공문 양식, 작성 예시 |
| 3. VCworks 접속 | 제출자료 정리용 계정으로 접속 | 계정 안내 메일 |
| 4. 파일 업로드 | VICS 데이터, 기존 평가사 제출자료, 첨부파일 업로드 | 업로드 가이드 |
| 5. 자동 생성 | 데이터 생성 실행 후 자동 생성 결과 확인 | 화면 가이드 |
| 6. 부족 항목 보완 | 표시된 누락값과 첨부파일 보완 | 입력 가이드, FAQ |
| 7. 제출 승인 | NICE P&I 또는 KIS자산평가 제출자료 생성 승인 | 제출 전 체크리스트 |
| 8. 평가 진행 | 평가기관 보완 요청 대응 | 보완 요청 처리 가이드 |
| 9. 후속 활용 | VICS 보고, 포트폴리오 관리, 영업보고서 등 확장 | 후속 활용 안내 |

### 11.3 VCworks 1년 무상 사용 지원 방향

VCworks 1년 무상 사용은 단순 데모 계정 제공이 아니라, 실제 공정가치평가 데이터를 넣고 평가기관 제출자료를 만든 뒤, 동일 데이터를 투자자산관리·보고·분석 업무에 재사용해 보는 방식으로 운영합니다.

- 대상 조합: 모태펀드 자조합 중 NICE P&I 또는 KIS자산평가에 의뢰하여 가치평가를 수행하는 조합(2026년도 신규 도입조합 대상)입니다. 평가기관이 안내한 관심 운용사와 내부 파일럿 운용사의 해당 조합을 우선 대상으로 합니다.
- 기간: 평가자료 제출 준비부터 1년 무상 사용 기간까지 지원합니다.
- 기본 범위: 공정가치평가 자료 생성, VICS 데이터 업로드, 부족 항목 보완, 제출자료 생성을 포함합니다.
- 확장 범위: BI·리포트 생성, 영업보고서 자동화, 투자·회수 관리, 가치평가 데이터 관리, LPworks·STworks 연계를 포함합니다.
- 운영 원칙: 처음에는 공정가치평가 제출에 필요한 범위로 시작하고, 실제 데이터가 정리된 뒤 활용 기능을 단계적으로 안내합니다.
- 후속 활용: 제출 이후 포트폴리오 관리, VICS 보고, LP 보고, 영업보고서, BI 활용과 LP·투자기업 대상 서비스 확장 가능성을 안내합니다.

## 12. 공동 안내 및 설명회 제안

### 12.1 고객 안내 방식

평가기관이 공정가치평가 자료 요청 메일을 발송할 때, VICS 데이터 기반 제출자료 정리 방식을 함께 안내하는 방식이 가장 현실적입니다.

- 안내문 문구 포함: “VCworks를 활용해 제출자료를 정리할 수 있습니다”라는 문구를 삽입합니다.
- 별도 링크 제공: VCworks 공정가치평가 신청 또는 안내 페이지 링크를 제공합니다.
- 퀵가이드 첨부: VICS 데이터 확보, 업로드, 부족 항목 보완 절차를 안내합니다.
- 문의 채널 분리: 평가 관련 문의는 평가기관, 시스템 사용 문의는 똑똑으로 분리합니다.

- 고객 후보 추천: 7월 파일럿 참여 가능성이 높은 운용사를 추천해 주실 수 있는지 확인합니다.
- 안내문 검토: 평가기관 명의 안내문에 포함 가능한 표현을 검토해 주실 수 있는지 확인합니다.
- 필수값 확정: 설명회 전 필수·보완·협의 항목 구분을 확정할 수 있는지 확인합니다.

- 설명회 참여: 평가기관 담당자가 자료 제출 기준 설명에 참여할 수 있는지 확인합니다.


## 13. 제안 결론

2026년 7월 공정가치평가 업무는 운용사와 평가기관 모두에게 새로운 운영 부담을 만들 수 있습니다. VCworks를 활용하면 운용사가 VICS 데이터와 기존 자료를 기반으로 제출자료를 구조화하고, 평가기관은 보다 표준화된 형태로 자료를 수신할 수 있습니다.

운용사 입장에서는 공정가치평가 자료 제출을 더 쉽게 준비할 수 있을 뿐 아니라, VCworks를 1년간 무상으로 사용하며 BI, 영업보고서 자동화, 투자·회수 관리, 가치평가 데이터화 등 주요 기능을 실제 데이터로 경험할 수 있습니다. 또한 LP는 LPworks를, 투자기업은 STworks를 활용할 수 있어 운용사와 이해관계자 간 자료 조회·보고·소통 업무까지 함께 개선할 수 있습니다. 이는 평가기관이 고객에게 제시할 수 있는 명확한 부가 혜택입니다.

NICE P&I와 KIS자산평가에는 동일한 기본 협력 구조를 제안드립니다. 초기에는 완전한 시스템 연동보다 7월에 실제 사용할 수 있는 제출자료 생성·검수·전달 흐름을 우선합니다. 이후 파일럿 결과를 바탕으로 평가기관별 제출 포맷, 고객 안내 방식, 전송 방식, VICS 보고 연계 가능성을 단계적으로 고도화할 수 있습니다.

똑똑은 2026년 7월 적용을 목표로 운용사 퀵가이드, VICS 데이터 확보 절차, VCworks 입력 화면, 평가기관 제출자료 생성 흐름, 파일럿 운영 지원을 준비하겠습니다.

## 1. VCworks 1년 무상 사용 고객 혜택

공정가치평가 대응 운용사에 제공하는 VCworks 1년 무상 사용 혜택을 정리한 문서입니다. 대상 조합, 주요 고객 효익, LPworks·STworks 연계 혜택, 평가기관 안내 메시지로 구성하며, 분량이 많아 별도 문서로 분리해 관리합니다.

→ [별첨 1. VCworks 1년 무상 사용 고객 혜택 (전문)](/assets/files/vics-fair-value-proposal/appendix-1-vcworks-benefits.pdf)

대상 조합은 모태펀드 자조합 중 NICE P&I 또는 KIS자산평가에 의뢰해 가치평가를 수행하는 조합(2026년도 신규 도입조합)입니다. 운용사는 공정가치평가 준비 과정에서 정리한 데이터를 바탕으로 VCworks를 1년간 무상 사용하며, BI·리포트, 영업보고서 자동화, 투자·회수 관리, 가치평가 데이터화 등을 실제 데이터로 검증할 수 있습니다. VCworks 사용 고객의 LP는 LPworks를, 투자기업은 STworks를 함께 활용할 수 있어 운용사·LP·투자기업 간 소통과 보고 업무까지 확장됩니다.

## 2. 데이터 사양서 및 평가기관 운영 기준

VCworks가 생성하는 평가기관 제출자료의 데이터 구조와, 평가기관이 그 자료를 수신·검수·회신할 때 적용하는 운영 기준을 함께 정의하는 문서입니다. 평가기관과 실무 협의할 때 데이터 종류, 컬럼, 필수 여부, 코드값, 생성·전송 방식, 검수·보완 절차를 명확히 하기 위해 작성하며, 항목 수가 많아 별도 문서로 분리해 관리합니다.

→ [별첨 2. 데이터 사양서 및 평가기관 운영 기준 (전문, ver.260602)](/assets/files/vics-fair-value-proposal/appendix-2-data-spec.pdf)

사양서는 데이터 생성·전송 방식(VCworks 원천 → 생성·확정 → 데이터 종류별 CSV 생성 → SFTP 업로드), 데이터 7종 파일 구성, 데이터별 컬럼 정의, 코드값·허용값, 주요 생성·검증 규칙, 평가기관 운영 기준(검수 체크리스트·보완 회신·역할 구분)으로 구성합니다. 별도의 평가기관 운영 매뉴얼은 본 문서에 통합했습니다. 현재 버전은 ver.260602이며 2026년 6월 9일 NICE P&I에 전달한 정의서를 기준으로 합니다. NICE P&I·KIS자산평가별 컬럼 매핑과 샘플 CSV는 평가기관 협의 후 보강합니다.

## 3. VICS 데이터 제공 요청 공문 양식 및 활용 가이드

운용사가 한국벤처투자(KVIC)에 VICS 보유 데이터를 요청할 때 사용하는 공문 양식과, 수령한 데이터를 VCworks로 가져오는 절차를 정리한 문서입니다. VICS 화면에서는 조합 목록 수준의 요약 데이터만 내려받을 수 있고, 조합·조합원·출자·투자자산의 상세 데이터는 공문 발송 시 VICS가 추출해 제공하므로, 이 공문 절차가 데이터 확보의 출발점입니다.

→ [별첨 3. VICS 데이터 제공 요청 공문 양식 및 활용 가이드 (전문)](/assets/files/vics-fair-value-proposal/appendix-3-vics-request-letter.pdf)

가이드는 데이터 확보 전체 흐름(공문 발송 → 상세 데이터 수령 → 똑똑 전달 → VCworks 마이그레이션 업로드·반영), 실제 발송 사례 기준 공문 작성 양식, 운용기관ID·조합목록 작성 팁, 수령 데이터와 마이그레이션 매핑(1차 조합·조합원·출자배분 / 2차 투자조건·피투자기업)으로 구성합니다. VICS 제공 상세 데이터의 항목·형식은 운용사별 공문 회신 결과에 따라 확정합니다.

## 4. 운용사 퀵가이드

운용사가 VCworks에서 공정가치평가 자료를 생성·검토·전송하는 방법을 안내하는 문서입니다. 평가 건 등록부터 데이터 생성, 검토·보완, CSV 전송까지의 사용 흐름과 화면별 안내, 상태별 작업을 정리합니다.

→ [별첨 4. 운용사 퀵가이드 (전문)](/assets/files/vics-fair-value-proposal/appendix-4-operator-quick-guide.pdf)

가이드는 사용 흐름 5단계(기본정보 등록 → 데이터 생성 → 검토·보완 → 데이터 전송 → 평가 진행), 데이터 검토 8개 탭, 화면별 안내(SCREEN 01~06), 상태별 작업으로 구성합니다. 상세 설명회 자료(`공정가치평가_킥오프미팅.pdf`)와 화면 상세 매뉴얼 링크를 함께 제공하며, 평가에 사용할 데이터를 VICS 역다운로드로 확보하는 절차는 별첨 3을 참조합니다.

## 5. VCworks 무상 프로모션 안내

공정가치평가 대응 모태출자펀드(VICS 자조합) 보유 운용사 및 기존 고객사를 대상으로 VCworks ERP를 1년간 무상 제공하는 프로모션입니다. 공정가치평가 자체는 NICE P&I·KIS자산평가가 수행하고, 운용사는 그 제출 자료를 VCworks에서 준비합니다.

→ [별첨 5. VCworks 무상 프로모션 안내 (전문)](/assets/files/vics-fair-value-proposal/appendix-5-promotion.pdf)

프로모션 혜택(VCworks 1년 무상, 신규고객 동일 혜택, 자료 생성부터 제출까지), 신청 대상·조건, 진행 절차(신청 → 접수·안내 → VICS 공문 발송 → 업로드·제출자료 생성)를 안내합니다. 신청은 [ddock.kr/promotion](https://ddock.kr/promotion)에서 받으며, 신청 운용사를 대상으로 VICS 데이터 확보(별첨 3)와 평가 자료 생성(별첨 4) 온보딩을 안내합니다.

## 6. 파일럿 피드백 양식 (작성 예정) : DSC 테스트 후 마이그레이션툴 개발완료 후 FO그룹 의견 필요

7월 파일럿에서 운용사, 평가기관, 똑똑이 확인한 내용을 같은 기준으로 회수하기 위한 양식입니다. 자동 생성률, 운용사 보완 항목, 평가기관 수신 가능 여부, 시스템 이슈, 개선 우선순위를 기록합니다.

구성 개요:

- 파일럿 기본정보: 운용사, 조합, 평가기관, 평가기준일
- 입력자료: VICS 데이터, 기존 평가자료, 첨부파일
- 자동 생성 결과: 영역별 자동 생성률, 누락 항목 수, 오류 항목
- 운용사 보완 항목: 운용사가 직접 입력한 항목과 소요 시간
- 평가기관 피드백: 수신 가능 여부, 누락 자료, 추가 요청, 포맷 수정 요청
- 시스템 이슈: 업로드 오류, 화면 불편, 성능 문제, 검증 오류
- 개선 우선순위: 7월 즉시 반영, 다음 버전 반영, 협의 필요
- 최종 판단: 계속 진행, 보완 후 진행, 별도 협의 필요

추후 보강:

- 실제 파일럿 고객별 결과
- 평가기관별 피드백 회신 양식
- 개선 일감 연결 기준
- 후속 설명회에 반영할 사례 정리

## 7. 설명회 자료 (작성 예정) : 6번 작업이 완료된 시점에 기존 1~6모든 자료를 합산해서 제작

7월 말 전후 파일럿 고객과 관심 운용사를 대상으로 진행할 공정가치평가 설명회 자료입니다. 제품 소개보다 공정가치평가 자료 준비 방법, 평가기관 제출자료 구성, VCworks 사용 흐름을 중심으로 구성합니다.

구성 개요:

- 공정가치평가 제도 변화와 운용사 준비사항
- NICE P&I·KIS자산평가 자료 요청 기준
- VCworks 활용 시나리오
- VICS 데이터 업로드와 자동 생성 흐름
- 부족 항목 보완 화면 설명
- 평가기관 제출자료 생성 시연
- 파일럿에서 확인된 자동 생성률과 부족 항목
- 평가 이후 VICS 보고, 포트폴리오 관리, 영업보고서 활용 가능성
- 파일럿 지원 대상, 기간, 신청 방법, 지원 범위
- 질의응답

추후 보강:

- 실제 시연 화면
- 파일럿 결과 수치
- 평가기관별 메시지
- 고객 사례
