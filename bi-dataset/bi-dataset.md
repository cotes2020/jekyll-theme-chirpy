## 집계 (Metric)
---

### 1. 투자금액 (KRW) {#metric-투자금액}

누적 투자금액입니다. 거래구분이 `투자`인 건의 거래금액을 모두 합산합니다.

```sql
SUM(CASE WHEN "거래구분" = '투자' THEN "거래금액" ELSE 0 END)
```

- **포맷**: `,d` (정수, 천 단위 콤마) · 통화 표기 KRW(앞)
- **주의**: 거래구분 시스템코드와 일치해야 합니다. 전환거래, 본계정인수의 추가 고려가 필요할 수 있습니다.

<br>

### 2. 투자잔액 (KRW) {#metric-투자잔액}

거래구분에 따라 투자원금이 더해지거나 차감되어 산출되는 현재 투자잔액입니다.

```sql
SUM(CASE
WHEN "거래구분" IN ('투자', '감액손실환입', '주식인수합병(+)', '본계정인수(+)')
THEN "거래원금"
WHEN "거래구분" IN ('유상감자', '감액손실', '청산', '회수', '단주대금', '주식인수합병(-)', '본계정인수(-)')
THEN "거래원금" * -1
ELSE 0 END)
```

- **포맷**: `,d` · 통화 표기 KRW(앞)
- **주의**: 거래구분 시스템코드와 일치해야 합니다.

<br>

### 3. 기여율 적용 투자금액 {#metric-기여율-적용-투자금액}

심사역의 총기여율을 반영한 투자금액입니다. (`투자금액 × 총기여율`)

```sql
SUM(CASE WHEN "거래구분" = '투자' THEN ("거래금액"*"총기여율"/100) ELSE 0 END)
```

- **포맷**: `,d`
- **주의**: 금액이 String으로 인식되는 버그가 재현될 경우 `CAST({sql} AS BIGINT)`와 같이 CAST 구문으로 감싸야 합니다.

<br>

### 4. 회수총액 (KRW) {#metric-회수총액}

누적 회수 총액으로, **회수원금과 회수손익의 합**입니다. 회수·단주대금·이자·현금배당·기타수익·유상감자 등 회수성 거래의 거래금액을 합산합니다.

```sql
SUM(CASE WHEN "거래구분" IN ('회수','단주대금','본계정인수(-)','이자','현금배당','기타수익', '유상감자') THEN "거래금액" ELSE 0 END)
```

- **포맷**: `,d` · 통화 표기 KRW(앞)
- **주의**: 거래구분 시스템코드와 일치해야 합니다.

<br>

### 5. 회수원금 (KRW) {#metric-회수원금}

누적 회수 원금입니다. 회수성 거래의 거래원금을 합산합니다.

```sql
SUM(CASE WHEN "거래구분" IN ('회수','단주대금','본계정인수(-)','이자','현금배당','기타수익','유상감자') THEN "거래원금" ELSE 0 END)
```

- **포맷**: `,d` · 통화 표기 KRW(앞)
- **주의**: 거래구분 시스템코드와 일치해야 합니다.

<br>

### 6. 회수손익 (KRW) {#metric-회수손익}

누적 회수손익입니다. 회수성 거래의 **처분손익금액과 외환차손익금액**을 합산합니다.

```sql
SUM(CASE WHEN "거래구분" IN ('회수','단주대금','본계정인수(-)','이자','현금배당','기타수익','유상감자') THEN "처분손익금액" ELSE 0 END)
+ SUM(CASE WHEN "거래구분" IN ('회수','단주대금','본계정인수(-)','이자','현금배당','기타수익','유상감자') THEN "외환차손익금액" ELSE 0 END)
```

- **포맷**: `,d` · 통화 표기 KRW(앞)
- **주의**: 거래구분 시스템코드와 일치해야 합니다.

### 7. 기여율 적용 회수금액 {#metric-기여율-적용-회수금액}

심사역의 총기여율을 반영한 회수금액입니다. (`회수금액 × 총기여율`)

```sql
SUM(CASE WHEN "거래구분" IN ('회수','단주대금','본계정인수(-)','이자','현금배당','기타수익', '유상감자') THEN ("거래금액"*"총기여율"/100) ELSE 0 END)
```

- **포맷**: `,d`
- **주의**: 금액이 String으로 인식되는 버그가 재현될 경우 `CAST({sql} AS BIGINT)`와 같이 CAST 구문으로 감싸야 합니다.

### 8. 감액 (KRW) {#metric-감액}

누적 감액입니다. `감액손실`은 더하고, `감액손실환입`은 차감하여 계산합니다.

```sql
SUM(CASE 
WHEN "거래구분" IN ('감액손실') THEN "거래금액"
WHEN "거래구분" IN ('감액손실환입') THEN "거래금액" * -1
ELSE 0 END)
```

- **포맷**: `,d` · 통화 표기 KRW(앞)
- **주의**: 거래구분 시스템코드와 일치해야 합니다.

### 9. 보유주식수 {#metric-보유주식수}

거래주식수를 기반으로 해당 시점의 보유주식수를 계산합니다. 취득성 거래(투자·무상증자·각종 전환 등)는 더하고, 처분성 거래(유상감자·회수·소각 등)는 차감합니다.

```sql
SUM(
CASE 
  WHEN "거래구분" IN (
    '투자', '무상증자', '액면분할', '주식증여', 
    '주식전환(+)', '워런트행사(+)', '전환사채전환(+)',
    'SAFE전환(+)', '주식인수합병(+)', 'Flip(+)', '본계정인수(+)'
  ) THEN COALESCE("거래주식수", 0)
WHEN "거래구분" IN (
    '유상감자', '본계정인수(-)', '무상감자', '주식병합', '주식소각',
    '회수', '주식전환(-)', '주식인수합병(-)', 'Flip(-)'
  ) THEN COALESCE("거래주식수", 0) * -1
ELSE 0
END
)
```

- **포맷**: `,d`
- **주의**: 거래구분 시스템코드와 일치해야 합니다.
